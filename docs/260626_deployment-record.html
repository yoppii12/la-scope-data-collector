作業ログ

01フロントエンドのビルド（Mac）

Mac のターミナルでフロントエンドをビルドし、RPi4 が配信できる静的ファイル（dist/）を生成した。

cd /Users/yutaromaruta/00_projects/la-scope-data-collector/frontend npm run build
02SSH 接続 — IP アドレスで接続不可
問題IP アドレス（192.168.10.104）で SSH 接続しようとしたところ Connection refused が返った。RPi4 の SSH が初期状態で無効になっていた。
対処： RPi4 に直接キーボード・モニターを接続し、SSH を有効化した。
sudo systemctl enable ssh sudo systemctl start ssh
その後、IP アドレスではなくホスト名で接続成功。
ssh pi@raspberrypi260624.local
03カメラ設定の確認

sudo raspi-config → Interface Options を確認したが、Legacy Camera / Camera の項目が存在しなかった。

結論： Raspberry Pi OS Bookworm ではカメラがデフォルトで有効のため、この手順は不要。

04ファイル転送（Mac → RPi4）

rsync で不要なフォルダを除外しながらプロジェクト一式を転送した。

rsync -av \ --exclude='venv' \ --exclude='node_modules' \ --exclude='backend/data' \ --exclude='.git' \ /Users/yutaromaruta/00_projects/la-scope-data-collector/ \ pi@raspberrypi260624.local:/home/pi/la-scope-data-collector/
05システムパッケージのインストール

RPi4 上でカメラライブラリと動画処理ソフトをインストールした。

sudo apt update && sudo apt install -y python3-picamera2 ffmpeg
06Python 仮想環境の構築

--system-site-packages を付けて仮想環境を作成。このフラグがないと apt でインストールした picamera2 が認識されない。

cd /home/pi/la-scope-data-collector python3 -m venv --system-site-packages venv source venv/bin/activate pip install -r backend/requirements.txt
07systemd サービスの登録

電源投入時に自動起動するよう systemd に登録・起動した。

sudo cp la-scope.service /etc/systemd/system/ sudo systemctl daemon-reload sudo systemctl enable la-scope sudo systemctl start la-scope
08サービス起動エラー — カメラ未検出
問題サービスが activating (auto-restart) を繰り返し、起動に失敗した。
エラー内容：
File "camera.py", line 120, in start self.picam = Picamera2() IndexError: list index out of range
原因： カメラケーブルが未接続のため Picamera2() がカメラを1台も検出できなかった。しかしコード上のフォールバック処理が不足しており、MockCamera に切り替わらずクラッシュしていた。
修正backend/app/camera.py の create_camera() を修正
カメラハードウェアが検出できない場合は MockCamera に自動フォールバックするよう変更した。
# 修正前 def create_camera(): if PICAMERA2_AVAILABLE: return RPiCamera() return MockCamera() # 修正後 def create_camera(): if PICAMERA2_AVAILABLE: try: if Picamera2.global_camera_info(): return RPiCamera() except Exception: pass return MockCamera()
修正後、ファイルを再転送してサービスを再起動したところ active (running) を確認。
09動作確認

Mac のブラウザから http://raspberrypi260624.local:8000 にアクセスし、LA-Scope アプリ画面の表示を確認した。

現時点ではカメラケーブル未接続のため MockCamera（テスト映像）で動作。カメラ接続後は自動的に実カメラ映像に切り替わる。
