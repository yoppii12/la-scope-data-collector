STEP 01 Mac でターミナルを開く

ターミナルとは、コンピュータにテキストで命令を送るアプリです

Mac のターミナルを開きます。

手順

キーボードの Command ⌘ + スペース を同時に押して Spotlight を開き、「ターミナル」と入力して Enter を押します。

💡
黒または白い画面に文字だけのウィンドウが開きます。これがターミナルです。
「見つからない場合」→ Finder → アプリケーション → ユーティリティ → ターミナル.app
✓
こうなれば成功： 下のような画面が開き、カーソルが点滅している
ターミナルの画面例 yourname@MacBook-Pro ~ % ← カーソルがここで点滅
✓
STEP 02 フロントエンドをビルドする

アプリの画面ファイルを RPi4 が読める形に変換します

ターミナルで以下の2つのコマンドを順番に実行します。1行ずつ貼り付けて Enter を押してください。

① プロジェクトフォルダに移動する 
cd /Users/yutaromaruta/00_projects/la-scope-data-collector/frontend
💡
cd は「フォルダを移動する」命令です。Enter を押しても何も表示されなければ成功です。
② ビルドを実行する 
npm run build

しばらく（10〜30秒）かかります。完了まで待ちましょう。

✓
こうなれば成功： 以下のような文字が表示される
成功時の表示例 ✓ built in 2.86s dist/index.html 0.76 kB dist/assets/index-xxx.js 478.34 kB
✓
STEP 03 RPi4 の IP アドレスを調べる

IP アドレスとは、ネットワーク上の「住所」です

Mac から RPi4 に繋ぐためには、RPi4 の「住所（IPアドレス）」が必要です。以下のどちらかの方法で調べてください。

方法 A：RPi4 の画面とキーボードが繋がっている場合

RPi4 のターミナルを開いて以下を入力します：

hostname -I
表示例 192.168.1.105 169.254.x.x ← 最初の数字が IP アドレス
方法 B：Mac のターミナルから調べる 
ping raspberrypi.local
表示例（Ctrl+C で止める） PING raspberrypi.local (192.168.1.105): 56 data bytes 64 bytes from 192.168.1.105 ...

表示された 192.168.1.xxx の形の数字が IP アドレスです。メモしておいてください。

💡
以降の手順では 192.168.1.XXX と書いてある部分を、メモした IP アドレスに置き換えて実行してください。
✓
こうなれば準備完了： 192.168.x.x 形式の数字が確認できた
✓
STEP 04 SSH で RPi4 に接続する

SSH とは、ネットワーク越しに RPi4 を遠隔操作する仕組みです

Mac のターミナルから、RPi4 に「リモート接続」します。

ssh pi@192.168.1.XXX

192.168.1.XXX は STEP 03 でメモした IP アドレスに変えてください。

💡
初回接続時に Are you sure you want to continue connecting (yes/no)? と聞かれます。yes と入力して Enter を押してください。
パスワードを入力する

pi@192.168.1.XXX's password: と表示されたらパスワードを入力します。

💡
デフォルトのパスワードは raspberry です。入力してもターミナルには何も表示されません（セキュリティのため）。正しく入力して Enter を押してください。
✓
こうなれば接続成功： プロンプトが pi@raspberrypi:~ $ に変わる
接続成功時の画面 Linux raspberrypi 6.x.x ... pi@raspberrypi:~ $ ← この表示に変わればOK
✓
STEP 05 カメラを有効化する

RPi4 のカメラ機能をONにします（初回のみ必要）

SSH 接続したままの状態（pi@raspberrypi と表示されている状態）で実行します。

sudo raspi-config

青い画面のメニューが表示されます。キーボードの矢印キーで操作します：

📋
操作手順：
1. Interface Options を選択 → Enter
2. Legacy Camera または Camera を選択 → Enter
3. <Yes> を選択 → Enter
4. <Ok> → Finish を選択
5. 再起動を聞かれたら <Yes>（再起動後に再度 SSH 接続が必要です）
✓
こうなれば成功： 設定が完了し、ターミナルのプロンプトに戻る（または RPi4 が再起動する）
✓
STEP 06 Mac からファイルを転送する

アプリのファイル一式を RPi4 に送ります

⚠ この手順は Mac のターミナルで実行します（RPi4 への SSH 接続は一旦閉じます）。

① SSH 接続を閉じる 
exit

プロンプトが yourname@MacBook-Pro に戻ることを確認してください。

② ファイルを転送する

以下のコマンドをコピーして、192.168.1.XXX をメモした IP アドレスに変えてから実行します。

rsync -av \
  --exclude='venv' \
  --exclude='node_modules' \
  --exclude='backend/data' \
  --exclude='.git' \
  /Users/yutaromaruta/00_projects/la-scope-data-collector/ \
  pi@192.168.1.XXX:/home/pi/la-scope-data-collector/
💡
rsync は「差分だけを転送する」コマンドです。ファイル数によって数秒〜1分ほどかかります。再度 SSH パスワードを求められます（raspberry）。
✓
こうなれば成功： ファイル名の一覧が流れてプロンプトに戻る
✓
STEP 07 RPi4 で必要なソフトをインストールする

再び SSH 接続し、カメラと動画のパッケージを用意します

① RPi4 に再接続する 
ssh pi@192.168.1.XXX
② システムを更新してパッケージをインストールする

以下のコマンドを実行します。インターネット接続が必要です。数分かかる場合があります。

sudo apt update && sudo apt install -y python3-picamera2 ffmpeg
💡
python3-picamera2 はカメラを操作するライブラリ、ffmpeg は動画を保存するソフトです。途中で Y/n? と聞かれたら Y を入力してください。
✓
こうなれば成功： エラーなく完了してプロンプトに戻る
✓
STEP 08 Python 仮想環境を構築する

アプリ専用の Python 実行環境を作ります

SSH 接続したまま（pi@raspberrypi の状態）で順番に実行します。

① プロジェクトフォルダに移動する 
cd /home/pi/la-scope-data-collector
② 仮想環境を作成する
⚠
--system-site-packages を必ず付けてください。 このオプションがないとカメラが認識されず、映像が映りません。
python3 -m venv --system-site-packages venv
💡
仮想環境とは「このアプリ専用の Python の部屋」です。他のソフトと干渉しないように分けて管理します。完了しても何も表示されません（正常です）。
③ 仮想環境を有効にする 
source venv/bin/activate
✓
こうなれば成功： プロンプトの先頭に (venv) が付く
有効化後の表示 (venv) pi@raspberrypi:~/la-scope-data-collector $
④ Python パッケージをインストールする 
pip install -r backend/requirements.txt
✓
こうなれば成功： Successfully installed ... と表示される
✓
STEP 09 自動起動サービスを登録する

RPi4 の電源を入れるだけでアプリが起動するように設定します

SSH 接続したまま実行します。

サービスを登録・起動する（4行まとめて実行可） 
sudo cp /home/pi/la-scope-data-collector/la-scope.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable la-scope
sudo systemctl start la-scope
起動状態を確認する 
sudo systemctl status la-scope
✓
こうなれば成功： Active: active (running) と緑色で表示される
成功時の表示 ● la-scope.service - LA-Scope Dataset Collector Loaded: loaded (/etc/systemd/system/la-scope.service; enabled) Active: active (running) since ... Main PID: 1234 (uvicorn)
💡
q キーを押すと表示を閉じてプロンプトに戻れます。
次回から RPi4 の電源を入れるだけでアプリが自動起動します。
✓

Mac のブラウザからアプリにアクセスします

Mac の Safari または Chrome を開き、アドレスバーに以下を入力します。

http://192.168.1.XXX:8000

192.168.1.XXX は STEP 03 でメモした IP アドレスに変えてください。

次回から RPi4 の電源を入れれば自動でアプリが起動します。

http://<RPi4 の IP アドレス>:8000

うまくいかないときは

SSH 接続時に「Permission denied」と出る ▼
パスワードが違います。デフォルトは raspberry です（小文字）。入力中は画面に何も表示されません。

パスワードを変更している場合は、変更後のパスワードを入力してください。
「ping raspberrypi.local」が反応しない ▼
RPi4 が同じ Wi-Fi に繋がっていない可能性があります。

RPi4 の画面で Wi-Fi の接続状況を確認してください。また、RPi4 の画面で hostname -I を直接実行して IP を調べる方法も試してみてください。
アプリは開けるがカメラ映像が黒画面のまま ▼
以下の2つを確認してください。

① カメラが有効になっているか（STEP 05 の操作）
SSH で接続して sudo raspi-config → Interface Options → Camera → Enable を確認。

② カメラの接続不良
RPi4 の電源を切り、カメラケーブルを差し直してから再起動してみてください。
ブラウザで「このサイトにアクセスできません」と出る ▼
① アプリが起動しているか確認する
SSH で接続して sudo systemctl status la-scope を実行し、active (running) と表示されているか確認。

② IP アドレスが変わっていないか確認する
STEP 03 の方法で IP を再確認してください。RPi4 を再起動すると IP が変わる場合があります。

③ ポートが塞がれていないか確認する
SSH で接続して sudo ufw allow 8000 を実行してみてください。
systemctl status で「failed」と表示される ▼
エラーの詳細を確認します。SSH で接続して以下を実行してください：

sudo journalctl -u la-scope -n 50

表示されたエラーメッセージをコピーして、サポートに共有してください。多くの場合、STEP 08 の --system-site-packages が抜けている問題です。

仮想環境を作り直す場合：
rm -rf venv
python3 -m venv --system-site-packages venv
