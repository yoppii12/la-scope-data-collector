LA-Scope Dataset Collector — Setup Guide

RPi4 アクセスポイント設定ガイド
Wi-Fiルーターなしで接続する

対象環境：Raspberry Pi 4 / Raspberry Pi OS Bookworm　|　作成日：2026-06-30

このガイドでは、RPi4 自身がWi-Fiを発信するアクセスポイントとして動作するよう設定します。設定後は外部のWi-Fiルーターが不要になり、PCをRPi4の「LA-Scope」Wi-Fiに繋ぐだけでアプリにアクセスできます。

設定手順
1
RPi4 に SSH接続する（現在のWi-Fi経由）

まず現在のWi-Fi環境で RPi4 に接続します。これが最後のWi-Fi経由でのSSH接続になります。

$ ssh pi@192.168.10.118
こうなれば成功
プロンプトが pi@raspberrypi260624:~ $ に変わればOKです。
2
ホットスポットを作成して自動起動設定を行う

以下のコマンドを実行します。ホットスポットの作成と、再起動後も自動で有効になる設定を一括で行います。

注意
このコマンドを実行すると RPi4 のWi-Fiが切り替わるため、SSH接続が切断されます。これは正常な動作です。 
pi@raspberrypi260624:~ $ sudo nmcli device wifi hotspot ifname wlan0 ssid "LA-Scope" password "lascope1234" && sudo nmcli connection modify Hotspot connection.autoconnect yes
こうなれば成功
Device 'wlan0' successfully activated... と表示され、その後SSH接続が切れればOKです。
3
PC を「LA-Scope」Wi-Fi に接続する

PCのWi-Fi設定を開き、「LA-Scope」というネットワークに接続します。

ネットワーク名（SSID）
LA-Scope
パスワード
lascope1234
こうなれば成功
Wi-Fiが「LA-Scope」に接続済みと表示されればOKです。
4
ブラウザでアプリにアクセスする

ブラウザで以下のURLを開きます。

http://10.42.0.1:8000
こうなれば成功
LA-Scope のカメラ画面が表示されればデプロイ完了です。
5
（任意）新しいIPアドレスでSSH接続する

今後 RPi4 に SSH接続する場合は、新しいIPアドレスを使います。

$ ssh pi@10.42.0.1
トラブルシューティング
?
「LA-Scope」Wi-Fiが表示されない

RPi4 を再起動してから再度Wi-Fiを確認してください。

# RPi4のモニター・キーボードが使える場合
pi@raspberrypi260624:~ $ sudo reboot
?
ホットスポットの設定を削除して元に戻したい

RPi4 にモニター・キーボードを繋ぐか、旧Wi-Fi経由でSSH接続できる環境で実行してください。

# ホットスポット接続を削除
pi@raspberrypi260624:~ $ sudo nmcli connection delete Hotspot

# 元のWi-Fiに再接続
pi@raspberrypi260624:~ $ sudo nmcli device wifi connect "元のSSID" password "元のパスワード"
注意事項： RPi4 をアクセスポイントモードで動作させている間、RPi4 自身はインターネットに接続できません。ソフトウェアのアップデートやパッケージのインストールが必要な場合は、一時的にホットスポットを停止して元のWi-Fiに戻す必要があります。
