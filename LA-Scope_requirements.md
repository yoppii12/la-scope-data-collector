# 要件定義書

**プロジェクト名：** LA-Scope Dataset Collector  
**作成日：** 2026年6月23日  
**作成者：** LAplust株式会社  
**バージョン：** 1.0.0

---

## 1. プロジェクト概要

### 1.1 目的

画像解析AIの教師データ収集を目的とした、Raspberry Pi 4上で動作する簡易Webアプリケーションを構築する。既設の顕微鏡にRaspberry Pi 4（以下RPi4）およびRPi HQカメラを装着し、PC上のブラウザからリモートで操作・撮影・データ管理を行えるシステムを実現する。

### 1.2 背景

- 顕微鏡観察画像をAI学習用データセットとして体系的に収集・管理する必要がある
- フィールドでの運用を想定し、軽量・シンプルな操作性を優先する
- 撮影データは日時ベースで自動整理し、後工程でのデータ活用を容易にする

### 1.3 スコープ

| 対象 | 内容 |
|------|------|
| **対象システム** | RPi4上で動作するWebアプリケーション（サーバー＋フロントエンド） |
| **対象ユーザー** | LAplust社内メンバー（研究・データ収集担当） |
| **対象外** | AI学習パイプライン自体、クラウドストレージ連携、ユーザー認証機能 |

## 2. システム構成

### 2.1 ハードウェア構成

| 機器 | 仕様 |
|------|------|
| コンピュータ | Raspberry Pi 4 Model B |
| カメラ | Raspberry Pi HQ Camera |
| ストレージ | microSDカード（容量は別途定義） |
| ネットワーク | Wi-Fi（RPi4内蔵） |

### 2.2 ネットワーク構成

```
[PC（操作端末）] ──── Wi-Fi ──── [Raspberry Pi 4（Webサーバー）]
                                        │
                                   [RPi HQカメラ]
                                        │
                                   [microSDカード（ストレージ）]
```

- PCとRPi4は同一Wi-Fiネットワーク上に接続する
- ユーザーはPC上のブラウザからRPi4の固定IPアドレスにアクセスする
- 例：`http://192.168.x.x:5000`

### 2.3 ソフトウェア構成（推奨）

| レイヤー | 技術スタック |
|---------|------------|
| OS | Raspberry Pi OS (64-bit) |
| バックエンド | Python / FastAPI または Flask |
| カメラI/F | `picamera2`ライブラリ |
| フロントエンド | React + MUI（Material UI） |
| 通信 | WebSocket（ライブビュー用） / REST API（撮影・ファイル操作） |
| タイムゾーン | Asia/Tokyo（JST、UTC+9） |

## 3. 機能要件

### 3.1 共通UI要素

#### 3.1.1 ヘッダー

- タイトル：**LA-Scope Dataset Collector** を常時表示
- 現在ページ名をサブタイトルとして表示

#### 3.1.2 フッター

- テキスト：**Powered by LAplust Inc.** を常時表示

#### 3.1.3 サイドバー

- 各画面へのナビゲーションボタンを配置
- 遷移先ページ一覧：

| ボタン名 | 遷移先 |
|---------|--------|
| ライブビュー | メイン撮影画面 |
| 履歴・閲覧 | 撮影履歴閲覧画面 |
| 設定 | システム設定画面（3.4節参照） |

---

### 3.2 メイン撮影画面

#### 3.2.1 ライブビュー

- RPi HQカメラの映像をリアルタイムでブラウザに表示する（MJPEG or WebSocketストリーミング）
- 表示解像度はブラウザウィンドウに合わせてレスポンシブに変化する
- カメラ未接続時はエラーメッセージを表示する

#### 3.2.2 撮影機能

| 機能 | 詳細 |
|------|------|
| **静止画撮影ボタン** | タップで1枚撮影。撮影成功時にトースト通知を表示 |
| **動画撮影ボタン** | タップで録画開始／停止をトグル切替。録画中はボタンを赤表示 |
| **撮影後プレビュー** | 直前に撮影した画像をサムネイル表示（動画は先頭フレーム） |
| **サンプルIDタグ付け** | 撮影前に任意のサンプルIDを入力できるフィールドを配置。入力値はファイル名（`YYMMDD-HHMMSS_<サンプルID>.jpg`）およびメタデータJSONに付与される |
| **撮影枚数カウンター** | 当日の撮影枚数（静止画 / 動画 別計）と累計枚数をライブビュー近傍に常時表示する |

#### 3.2.3 インターバル撮影

- 設定した時間間隔（秒単位）で静止画を自動連続撮影する
- 開始・停止ボタンでトグル制御。動作中はインジケーターを表示する
- 撮影間隔・撮影枚数上限を設定画面（3.4節）で指定できる
- インターバル撮影中も通常の手動撮影ボタンは有効とする

#### 3.2.4 フォーカスアシスト

- ライブビュー上にグリッドライン（3×3）を重畳表示するトグルボタンを配置する
- ピーキング表示（エッジ強調）のON/OFFを切り替えられるボタンを配置する
- フォーカスアシスト表示は撮影ファイルには影響しない（オーバーレイのみ）

#### 3.2.5 ストレージ表示

- microSDカードの**空き容量 / 総容量**をプログレスバーで表示
- 残量が総容量の20%を下回った場合は警告色（アクセントカラー）で強調表示

---

### 3.3 履歴閲覧画面

#### 3.3.1 ファイル管理ルール

| 項目 | 仕様 |
|------|------|
| フォルダ構成 | 日付別フォルダ `YYMMDD/`（例：`250623/`） |
| 静止画ファイル名 | `YYMMDD-HHMMSS.jpg`（例：`250623-143022.jpg`） |
| 動画ファイル名 | `YYMMDD-HHMMSS.mp4`（例：`250623-143055.mp4`） |
| タイムゾーン | JST（日本標準時）に基づくタイムスタンプ |

#### 3.3.2 ファイル一覧表示

- 日付フォルダをカード形式で一覧表示（最新順）
- フォルダ展開で当該日付の静止画・動画をサムネイル一覧表示
- ファイルタイプ（画像 / 動画）をアイコンで識別表示

#### 3.3.3 ファイル操作

| 機能 | 詳細 |
|------|------|
| **プレビュー** | クリック/タップで画像拡大表示またはインライン動画再生 |
| **個別ダウンロード** | ファイル単体をブラウザにダウンロード |
| **一括エクスポート（ZIP）** | フォルダ単位または全ファイルをZIPにまとめてダウンロード |
| **CSVエクスポート** | 撮影履歴（日時・ファイル名・サンプルID・観察条件等のメタデータ）をCSV形式で出力。フォルダ単位または全期間を選択可能 |
| **削除** | 確認ダイアログ表示後にファイル削除（誤操作防止） |

#### 3.3.4 ストレージ表示

- メイン画面と同様にSDカードの空き容量を表示する

---

### 3.4 システム設定画面

| 設定項目 | 詳細 |
|---------|------|
| **カメラ設定** | 解像度・フレームレート・露出・ホワイトバランスの調整 |
| **インターバル撮影設定** | 撮影間隔（秒）・撮影枚数上限を設定する。設定値はアプリ再起動後も保持する |
| **アノテーション補助** | 撮影時に付与する「サンプルID」「観察条件」等のデフォルト値を設定しJSONで保存 |
| **ネットワーク情報** | RPi4の現在のIPアドレスと接続状態を表示。サイドバーにも常時表示する |
| **ストレージ管理** | 古いファイルの日付指定一括削除・SDカードの使用状況詳細表示 |

## 4. 非機能要件

### 4.1 パフォーマンス

| 項目 | 目標値 |
|------|--------|
| ライブビュー遅延 | 1秒以内（Wi-Fi同一セグメント内） |
| 静止画撮影レスポンス | 2秒以内（シャッター操作から保存完了まで） |
| 画面読み込み | 3秒以内（初回アクセス） |

### 4.2 信頼性

- RPi4の電源再投入後、Webサーバーが自動起動すること（systemdサービス登録）
- 撮影中にネットワーク接続が一時切断されても、録画データが破損しないこと
- SDカード満杯時は撮影を停止し、ユーザーに明確にエラー通知すること

### 4.3 セキュリティ

- 同一LAN内からのみアクセス可能とする（外部公開不可）
- 初期フェーズではユーザー認証は対象外とするが、将来的な追加を考慮した設計とする

### 4.4 保守性

- バックエンドとフロントエンドの役割を明確に分離する
- カメラライブラリ（picamera2）はバックエンドのみで扱い、フロントエンドはREST API / WebSocketのみで通信する

## 6. データ仕様

### 6.1 ファイル形式

| データ種別 | フォーマット | 備考 |
|-----------|------------|------|
| 静止画 | JPEG（.jpg） | 最大解像度：4056×3040px（HQカメラ最大） |
| 動画 | MP4（H.264） | フレームレート：30fps（設定可変） |
| メタデータ（提案） | JSON（.json） | 静止画と同名で同フォルダに保存 |

### 6.2 メタデータ構造（提案）

```json
{
  "filename": "250623-143022.jpg",
  "captured_at": "2025-06-23T14:30:22+09:00",
  "sample_id": "",
  "note": "",
  "camera_settings": {
    "resolution": "4056x3040",
    "exposure": "auto",
    "white_balance": "auto"
  }
}
```

## 7. 制約・前提条件

- RPi4とPC間のWi-Fi通信速度が動画ストリーミングに十分であること（推奨：5GHz帯）
- microSDカードはFAT32またはexFAT形式でフォーマット済みであること
- `picamera2`ライブラリがRPi4にインストール済みであること
- Node.js / npmがRPi4にインストールされているか、フロントエンドはビルド済みの静的ファイルとして配信すること
